# 聊天输入框：让用户更容易给 Agent 下清楚的指令

## 它解决什么问题

输入框是 Agent 的入口。空输入还能发送、菜单挡住文字、运行状态不清楚，都会让用户误以为指令没有生效。

## 布局原则

一个圆角容器里放输入区和底部工具栏：左侧放添加动作和权限，右侧放运行状态、模型、思考级别和发送按钮。窄窗口时压缩间距，长模型名截断，菜单向上打开。

## 行为要保持简单

- Enter 发送，Shift+Enter 换行；
- 空白输入不能发送；
- 发送后清空并恢复最小高度；
- 多行自动增高但有上限；
- 流式运行时显示排队状态；
- 同时只开一个下拉菜单，点击外部关闭。

添加按钮如果还没有附件能力，只作为入口，不要假装已经支持附件。

## 组件边界

输入框读取现有 store 的模型、权限、思考级别和运行状态，不要在组件里复制一套 Agent 状态。它只收集输入、触发发送和展示状态。

## 可访问性

按钮要有 label，菜单要有展开状态，键盘可以操作；状态不能只靠颜色；浅色和深色主题都使用现有主题变量。
