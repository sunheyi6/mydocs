# 失败教训：记住“不要再怎么做”

## 它解决什么问题

普通记忆记录“项目是什么、做过什么”，却没有记录“为什么失败”。于是 Agent 下一次又会用同样的错误参数、同样的跨平台命令和同样的重试方式。

## 什么时候记教训

- 工具执行失败；
- 用户明确纠正 Agent；
- run 以 error 或 blocked 结束；
- Goal 验证连续两次得到相同失败输出。

不是每个非零退出码都值得永久保存，只有能复用的原因才形成教训。

## 一条好教训只有三件事

1. 问题：当时想做什么，哪里失败？
2. 根因：为什么失败？
3. 正确做法：下次先检查什么、怎么做？

再加上工具、相关文件和关键词，方便搜索。教训要短，详细代码回源码看。

## 为什么异步提取

教训提取不是当前任务的主路径。运行先结束，轻量模型后台整理；提取失败不能影响主任务，也不能让 Worker 崩溃。

## 为什么按需搜索

全部教训塞进 System Prompt 会变长，还会把无关错误带进当前任务。更好的做法是提供只读的 `search_lessons`，工具失败时最多追加一行提醒，只有相关教训才放全文。

## 去重和限流

同一错误可能重复出现，所以要做相似度去重、同一 run 限条数、同类型错误限频，并设置总数量上限。否则教训系统会变成错误的复制机。

## 和记忆的区别

- Memory：以后要记住的正向事实。
- Lesson：以后不要重复犯的负向经验。
