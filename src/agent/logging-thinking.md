# 运行事件日志：聊天结果之外，还要知道 Agent 做过什么

## 为什么要记

最终回答无法说明模型第几轮开始、调用了什么工具、哪里失败、为什么继续、为什么停止。没有这些证据，问题只能靠猜。

## 为什么是 JSONL

一行一条 JSON，可以边运行边追加；中途崩溃时，前面写入的事件还在；新增字段可以做成可选，旧日志仍能读取。

每个 run 一个文件，首行记录协议版本和创建时间。

## 记录四层事实

- Run：开始、完成、失败、中止和恢复。
- Turn：模型请求、轮次、耗时、首 token 延迟、token 和 stop reason。
- Tool：工具名、调用 ID、参数摘要、成功失败、退出码和截断状态。
- Content：text、thinking 和引导片段，用于重建过程。

Goal 还要记录每轮验证和最终状态，否则只知道“重试了几次”，不知道为什么重试。

## 日志边界

- 输出要截断，完整内容按需读取。
- 参数要脱敏，不能把密钥原样写入。
- 日志失败不能拖垮 Agent 主流程。
- 新字段尽量可选，读取旧文件时跳过未知字段。

## 日志有什么用

调用链面板、Token 统计、异常恢复、失败教训提取，都应该使用同一份事件事实，不要各自重新猜一套状态。
