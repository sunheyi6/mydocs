# Write 工具：安全地新建或整体覆盖文件

## 它解决什么问题

新建文件或整体重写文件时，write 比 edit 更合适。但整体覆盖的风险更大：路径错了会写错地方，和另一个工具并行还可能互相覆盖。

## 写入流程

1. 校验路径在工作目录内。
2. 校验内容存在。
3. 对目标文件进入 mutation queue。
4. 读取旧内容（如果有）。
5. 创建目录并写入。
6. 比较旧、新内容，返回增删行数。

## 为什么要读旧内容

为了计算本次真实变化。新文件是 `+N/-0`，覆盖文件按逐行差异统计。UI 不应该只显示文件总行数。

## 和 edit 怎么配合

同一文件必须串行，不同文件可以并行。写入失败要返回路径和原因，不能只返回一个模糊的 I/O error。

## 什么时候用

- 新建文件：write。
- 整体生成小配置：write。
- 局部修改已有代码：edit。
- 不确定原内容：先 read。
