# 缓存：重复计算可以省，但事实变化不能装作没变

## 为什么需要缓存

Agent 会重复读同一个文件、重复构造相同的上下文前缀、重复加载会话。每次都重新做会增加延迟和成本。

缓存的前提是：能判断结果还有效。不能确认有效，就不要缓存。

## Agent 里常见的缓存

- 文件内容缓存：键包含路径和文件修改时间；文件被改后必须失效。
- 会话展示缓存：先缓存最近消息快照，完整历史后台补齐。
- 前缀缓存：保持 System Prompt 和历史前缀稳定，帮助模型 API 复用计算。
- 搜索文件行缓存：同一次搜索里同一文件只读一次。
- 工具结果缓存：只有明确只读、参数相同且环境未变时才适合。

## 哪些东西不能随便缓存

- bash 命令结果：环境、时间和文件可能已变。
- 写入工具结果：写操作改变了事实，后续读取必须看到新内容。
- 带权限和用户身份的结果：不能跨 session 复用。
- 包含敏感数据的结果：缓存位置和生命周期要受控。

## 缓存失效比缓存命中更重要

常见失效原因：文件被写入、工作目录改变、权限改变、模型参数改变、会话切换、TTL 到期。写操作完成后要主动清理相关读缓存。

## 设计经验

- 先定义缓存键，再谈缓存策略。
- 缓存命中要可观测，否则排障时不知道读到的是旧数据还是新数据。
- 不要为了“命中率”牺牲正确性。
- 对用户可见的会话切换，缓存只优化显示，不能让 Worker 丢完整上下文。
