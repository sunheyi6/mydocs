# Bash 工具：让 Agent 能跑命令，但不能被命令拖垮

## 它解决什么问题

Agent 需要跑测试、构建和脚本，但命令可能输出几十万字符、长时间不结束，或者启动子进程后父进程死了子进程还在跑。

所以 bash 必须是受控执行器，不是简单的 `spawn(command)`。

## 输出为什么保留尾部

编译错误、测试汇总和构建最终状态通常在最后。只保留开头会把最有用的信息截掉。

当前思路是保留最后约 2000 行或 50KB。超出时返回尾部、明确提示被截断，并把完整 stdout/stderr 保存到临时文件，Agent 需要时再 read。

## 超时、溢出和停止

- 默认超时，最大超时不能无限放大；
- 输出超过约 200KB 时停止进程，保护内存；
- 用户停止任务时，要清理后台命令；
- 超时和溢出都要返回清楚原因，而不是只返回空结果。

## 为什么杀进程树

`npm run build` 可能启动 webpack。只杀父进程会留下孤儿进程。Windows 使用 `taskkill /T`，Unix 使用进程组或 PID 兜底，目标是清理整个命令树。

## 安全边界

危险命令黑名单只能挡住常见写法，不能当成完整安全方案。还要配合权限模式、用户确认或沙箱。命令通过字符串检查，不代表绝对安全。

## Agent 使用经验

- 构建失败先看输出尾部；没有结论再读完整日志。
- 命令失败不要原样重复，先判断是代码、路径还是环境问题。
- 开发服务器用后台模式，并通过后台进程状态管理。
