# 调用链面板：聊天看结论，轨迹看全过程

## 为什么需要单独的调用链

聊天区需要干净，但排查问题时必须看到用户请求、每轮模型、工具参数、工具结果、失败和结束原因。把所有细节都塞聊天里会很吵，只保留最终回答又没有证据。

## 两个视图分工

- 聊天区：最终回答和轻量进度。
- 调用链：完整执行大纲和可展开细节。

摘要行显示轮次、模型、耗时、token、工具数和状态；详情默认折叠，正在运行的项目突出显示。

## 为什么先在 Renderer 整理

已有 ChatMessage、ToolCall、TurnDetail 和 RunEvent。先写一个 view model 组织这些数据，比马上改 Worker 协议风险小，也更容易兼容旧会话。

旧数据缺少退出码或行数时显示“暂无”，不能伪造事实。
