# 悬浮面板：把 Agent 的外部状态放在手边

## 为什么需要它

用户在等待 Agent 时，想知道当前分支有没有修改、后台命令是不是还在跑、已经跑了多久。放在聊天正文里会打断阅读，完全隐藏又会让用户失去控制感。

## 折叠和展开

折叠态只显示分支、是否有变更和展开按钮；展开后显示 Git 变更、最新后台命令、运行状态、耗时和运行中进程数。

默认折叠是为了不遮挡聊天，不代表隐藏事实。点击外部、再次点击和键盘操作都应能收起。

## 数据来源

- Git 状态来自已有 Git bridge。
- 后台命令来自 started/completed 事件和共享进程集合。
- 最新命令按开始时间选择，不因旧命令晚结束而改变。

不是 Git 仓库或查询失败时显示安全空状态，不影响 Agent。

## 经验

- 状态不要只靠颜色，要有文字或图标。
- 长命令截断显示，但保留完整 title。
- 多个组件消费进程事件时，订阅必须共享。
- 只有总变更数时不能伪造新增和删除，必须有明确字段。
