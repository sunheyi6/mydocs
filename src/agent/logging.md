# 日志：每条记录都要能回答一个问题

## 先问日志用来干什么

Agent 日志主要不是给模型看的，而是给用户、开发者和恢复逻辑看的。常见问题包括：

- 这次运行有没有开始和结束？
- 模型请求慢在哪里？
- 调了哪个工具？参数是什么？
- 工具失败后有没有继续？
- 为什么最后停了？

## 记录事实，不记录猜测

好的日志是“tool `grep` 返回 exit code 1”；不好的日志是“模型可能没找到代码”。后者应该由排障者根据事实判断。

## 日志分层

- 生命周期日志：run 开始、完成、失败、中止。
- 模型日志：请求时间、模型、token、stop reason。
- 工具日志：开始、结束、参数摘要、结果和错误。
- 状态日志：规划、压缩、引导、Goal 验证和完成判断。

分层后既能看总览，也能深入某个工具。

## 生产环境注意

- 开发环境可以有 debug，生产环境默认 info/warn。
- 大输出必须限制大小。
- 用户路径和命令可能包含敏感信息，要脱敏。
- 日志写入应尽量追加，避免每次改写大文件。

## 经验

日志不是越多越好。保留能还原因果链的事件，其他内容按需从原文件或工具再次获取。
