# Token 用量：显示同一份事实，不造第二个计数器

## 为什么显示

Agent 运行时间长、上下文大时，用户需要知道消耗情况。Token 是成本和上下文压力的信号，但它不应该干扰主要任务。

## 只使用一个来源

顶部和底部都读取 Agent store 中已有的累计 token。不要在 UI 重新累加，否则流式事件延迟或重复时会出现两个数字。

## 显示规则

- 大于 0 时显示带千分位的数量，例如 `1,234 tokens`；
- 0 时隐藏，避免增加无用噪音；
- 只做格式化，不在组件里计算累计值。

## 经验

Token 展示是状态投影，不是统计系统。累计逻辑放 Worker/Agent，日志保存完整数据，UI 只展示当前口径。
