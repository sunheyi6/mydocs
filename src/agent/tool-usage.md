# 工具使用：先取证，再行动

## 工具怎么选

- 不知道文件名：`glob`。
- 知道关键词但不知道位置：`grep`。
- 要看真实内容：`read`。
- 局部改代码：`edit`。
- 新建或整体覆盖：`write`。
- 跑测试、构建和脚本：`bash`。

工具不是越多越好。先用最小动作取得事实，再决定下一步。

## 推荐工作顺序

`搜索位置 → 读取相关内容 → 选择最小修改 → 执行验证 → 根据结果继续`

不要没读文件就改，也不要为了“确认一下”重复读大量无关文件。

## 读工具结果

工具结果是新的事实，不是模型建议。成功结果要看内容，失败结果要看原因；截断结果要意识到不完整。需要更多信息时再精确读取，不要把 limit 盲目调大。

## 并行的边界

互不依赖的只读操作可以并行，例如同时 grep 两个目录。写同一个文件必须串行；依赖前一个结果的操作不能提前并行。

## 修改后必须验证

至少根据任务运行相关类型检查、测试或构建。验证失败后先理解错误，不要原样重复同一命令。环境错误和代码错误要区分。

## 权限和安全

只读工具风险低，写入和 bash 需要遵守权限模式。UI 上隐藏按钮不是安全措施，真正的限制必须在 Worker 和工具层执行。
