# 流式进度：摘要短一点，动作事实保留下来

## 它解决什么问题

模型中间话很多，全部显示会淹没聊天；完全隐藏又让用户不知道 Agent 是否在工作。

## 当前做法

- 模型用最多几行的短摘要说明进度。
- 工具名、文件路径、命令和失败状态结构化展示。
- 当前运行项高亮，已完成项可折叠。
- 最终回答单独显示一次。

摘要只影响 UI，不修改原始消息、blocks 或日志。需要完整过程时，用户可以展开或查看调用链。

## 为什么不用另一次 AI 摘要

额外请求会增加延迟、费用和失败点。前端预览和展开已经能满足快速浏览，同时保留全文。

## 不能隐藏的事实

工具失败、正在运行的命令、目标文件、截断提示和最终结果都要保留。状态不能只靠颜色表达。
