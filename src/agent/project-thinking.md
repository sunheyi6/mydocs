# 项目信息：Agent 先知道自己在哪儿

## 为什么要单独提供

Agent 经常需要回答日志在哪、配置在哪、项目如何构建。如果每次靠猜路径，开发版和正式版、项目目录和用户目录很容易混淆。

项目信息文档是事实表，不负责讲所有设计。

## 至少要有这些事实

- 当前项目根目录和工作目录；
- 开发版、正式版的数据目录；
- 会话、运行日志、Token、记忆和教训路径；
- 启动、检查、构建命令；
- 进程之间如何通信；
- 权限模式和工具边界。

## 为什么要区分开发和正式环境

两个版本共用 userData 会导致单实例锁冲突、互相顶窗口、读错会话。应用身份必须在最早阶段确定，早于第一次获取 userData 路径。

## 使用优先级

回答路径问题时优先看项目事实和实际日志，再看代码，最后才根据经验推测。推测必须明确说是假设。

## 常见错误

- 把项目目录和应用数据目录混为一谈。
- 只改 Renderer，忘了 Main、Worker 和 preload 的路径约定。
- 开发版写进生产目录。
- 文档写了旧命令，Agent 继续照旧执行。
