# 记忆系统：记住长期有用的事实，不保存整段聊天

## 为什么需要记忆

每次会话都从零开始，Agent 会反复询问项目技术栈、目录约定和用户偏好。把全部聊天历史都带上又太大，还会混入过时细节。

记忆要保存的是以后还会用到的事实。

## 三层记忆

- 全局：用户长期偏好。
- 项目：技术栈、目录、构建和代码约定。
- 会话：某次任务的重要决定、结果和未完成事项。

范围越窄，越应该优先服务当前任务。

## 两种来源

- 自动记录：会话结束后异步提取关键事实。
- 手动添加：用户明确说“记住这个规则”时保存。

自动提取不确定时宁可不记。一次性的临时猜测如果被写成长期事实，会比不记更危险。

## 为什么用文件和 Markdown

本地 Markdown 易检查、可备份、可删除，也不会为了记忆额外引入云服务。代价是检索能力有限，所以用关键词和相似度混合检索，并按时间和范围调整优先级。

## 检索不要太多

记忆按需检索，只把少量相关内容放进当前上下文；无关记忆会干扰模型。UI 最好显示引用，用户知道回答参考了什么。

## 记忆会过期

“项目使用 Vue”可能长期有效；“正在修复某个 bug”很快过期。要记录时间，允许新事实覆盖旧事实，也要允许用户查看、修改和删除。

## 常见错误

- 把整个会话当成一条记忆。
- 自动把中间猜测写成事实。
- 每轮把所有记忆注入 System Prompt。
- 不处理冲突和过期内容。
