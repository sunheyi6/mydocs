# 工具调用：模型想做什么，程序负责安全执行

## 它解决什么问题

模型只能生成文字，不能自己读磁盘、改文件和跑测试。工具调用把模型的动作请求交给程序执行，再把结果放回上下文。

基本循环是：

`模型决定动作 → 校验参数 → 执行工具 → 返回结果 → 模型决定下一步`

## 工具接口要统一

每个工具都需要名称、说明、参数 Schema、执行函数和统一结果。结果至少包括调用 ID、工具名、成功/失败和给模型看的输出；给 UI 的文件统计、命令退出码放在结构化 `details`。

## 执行时要防什么

- JSON 解析失败不能让循环崩溃。
- 必填参数缺失时不要执行，返回可行动错误。
- 常见类型错误可以有限转换，但不能任意猜。
- 同一文件的 write/edit 要串行。
- 工具输出要限长，完整内容按需读取。

## Registry 和 MCP

Registry 统一注册、查找、执行和生成工具定义。MCP 工具用 `mcp__server__tool` 命名，避免不同服务器重名，也让来源清楚。

## 工具失败后怎么办

错误信息要告诉模型下一步，例如“文件不存在，请用 glob 查找”，而不是只给 `ENOENT`。模型如果重复同一失败调用，Agent Loop 应该提醒、换思路或触发断路器。

## UI 的边界

Worker 产生工具事件，Renderer 根据结构化数据展示。UI 不能从一段人类可读文本猜增删行数、退出码或成功状态。
